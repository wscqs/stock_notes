# 笔记详情页删除态导航栏按钮设计

## 背景

笔记详情页（`lib/app/modules/noteedit`）目前无论笔记是否已在删除列表，都显示同一组导航栏按钮。当用户从删除列表进入笔记详情时，仍然看到“分享 / 标签 / 收藏 / 删除 / 保存”按钮，其中“删除”会把笔记从正常状态移到删除列表；但已经在删除列表的笔记需要“恢复”或“永久删除”。

## 目标

根据笔记的删除状态，动态显示不同的导航栏按钮：

- **非删除笔记**：保持现有按钮（预览 / 分享 / 标签 / 收藏 / 删除 / 保存）。
- **已删除笔记**：只显示“恢复”和“删除（永久删除）”。

## 方案

采用方案 A：干净分组。

### 行为定义

| 状态 | 导航栏按钮 | 行为 |
|------|-----------|------|
| `opDelete == false` | 预览、分享、标签、收藏、删除、保存 | 与现有逻辑一致 |
| `opDelete == true` | 恢复、删除 | 恢复：将 `opDelete` 设为 `false` 并返回；删除：调用 `db.deleteNote` 永久删除并返回 |

### 不变更点

- 二次确认弹窗：永久删除不需要二次确认（用户明确选择）。
- 返回拦截逻辑：保持不变。
- 笔记内容区：删除态仍然只读，不可编辑。

### 涉及文件

1. `lib/app/modules/noteedit/controllers/noteedit_controller.dart`
   - 新增 `clickRestore()` 方法：将 `localData.value!.opDelete` 设为 `false` 并更新数据库，返回上一页。
   - 保持 `clickOpDelete()`：当 `opDelete` 为 `true` 时执行永久删除。
2. `lib/app/modules/noteedit/views/noteedit_view.dart`
   - 根据 `controller.localData.value?.opDelete` 判断渲染哪一组按钮。
   - 已删除笔记只渲染“恢复”和“删除”两个 `IconButton`（或 `TextButton`）。

### 文本键

- 恢复：`TextKey.huifu.tr`
- 删除：`TextKey.delete.tr`

## 验收标准

1. 从正常列表进入笔记详情，导航栏显示原有按钮。
2. 从删除列表进入笔记详情，导航栏只显示“恢复”和“删除”。
3. 点击“恢复”后，笔记回到正常列表，页面返回。
4. 点击“删除”后，笔记被永久删除，页面返回。
5. 删除态下不显示保存、分享、标签、收藏、预览按钮。

## 备注

- 当前 `clickOpDelete()` 已兼容软删除和永久删除两种逻辑，无需大改。
- 恢复操作直接更新数据库并返回，不保存当前编辑器内容（删除态只读，无内容变更）。
